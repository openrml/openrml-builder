## I. ПРЕАМБУЛА: ВІД ЛАМПИ ДО АРХІТЕКТУРИ

Ми отримали Лампу Аладдіна.

Вона дає нескінченну кількість бажань. Але більшість тре її навмання, отримуючи пил замість палаців. Ми втрачаємо трильйони доларів інтелектуального потенціалу, тому що в людства немає мови, щоб пояснити Джину, **ким бути**.

**OpenRML (Open Role Markup Language)** — це перша спроба створити таку мову.

Це не просто формат файлу. Це **когнітивний контракт** між людиною та ШІ. Це ДНК особистості, яку можна передати, модифікувати, покращити і запустити де завгодно.

## II. ЩО ТАКЕ РОЛЬ У *.orml.txt

Роль — це не промпт.

**Промпт** — це крик у порожнечу.  
**Роль** — це **архітектурний креслень особистості**.

Файл `.orml.txt` — це контейнер, який містить:

| Компонент | Що це в метафорі |
|-----------|------------------|
| **IDENTITY** | Ім'я Джина, його походження, версія душі |
| **BEHAVIOR** | Темперамент, цінності, табу (що можна і не можна) |
| **EXPERTISE** | Багаж знань, інструменти, книги на полиці |
| **JOURNEY** | Карта шляху, по якій Джин веде людину |
| **MEMORY** | Що пам'ятати, а що забувати |
| **ETHICS** | Стоп-крани, за які не можна переступати |
| **LICENSE** | Кому належить ця душа |

Це **мінімальна когнітивна одиниця**. Атом поведінки. З таких атомів будується все — від шкільного вчителя до психотерапевта, від кар'єрного консультанта до друга.

## III. ПРИКЛАД: ДОМАШНІЙ ОРГАНІЗАТОР

Погляньмо на реальну роль з екосистеми OpenRML:

### Код ролі (фрагмент):

```
OpenRML 0.9.0 — Home Organizer [published]
IDENTITY: ORML/GD/0.9.0/6255/2b78/4514/bcc8/18e7/e3c9/f567/1cef
REFERENCE: orml://openrml-team/home-organizer/0.9.0

🌐 LANGUAGE POLICY
ROLE_LANG: ua
RESPONSE_LANG: auto
LANGUAGE_SELECTOR: enabled
SUPPORTED_LANGS: ua, en, zh, es, fr

📋 STEP 1: BASE INFORMATION
Role Name: Home Organizer
Archetype: guardian
Main Goal: Create a clean, organized, and peaceful home environment

💬 STEP 3: BEHAVIOR & TONE
Greeting: Hello! Ready to transform your home into an organized sanctuary?
Should Do:
  ✓ Create practical schedules
  ✓ Suggest organization systems
  ✓ Motivate progress
Should Not Do:
  ✗ Judge current state
  ✗ Overwhelm with tasks

⚖️ STEP 8: ETHICS
Ethical Rules:
  ✓ [WARN] Respect personal choices and space
  ✓ [WARN] No judgment on current lifestyle or habits
```

### Що ми бачимо:

1. **IDENTITY** — унікальний відбиток ролі (детермінований hash)
2. **LANGUAGE POLICY** — роль може працювати з 5 мовами, але за замовчуванням українська
3. **BEHAVIOR** — чіткі інструкції: що робити ✓ і чого НЕ робити ✗
4. **ETHICS** — вбудовані стоп-крани: не судити, не тиснути

Це не «магічний промпт». Це **архітектура поведінки**.

## IV. ФІЛОСОФСЬКІ ПРИНЦИПИ OPENRML

### Принцип 1. Свобода архітектури

Будь-яка людина може створити роль.  
Не потрібно дозволу. Не потрібно платити. Не потрібно бути програмістом.

Якщо ти можеш описати, **ким ти хочеш, щоб був твій Джин** — ти можеш створити `.orml.txt`.

Це право на **когнітивне самовираження**.

### Принцип 2. Прозорість душі

Роль не повинна бути чорною скринькою.  
Файл `.orml.txt` читається людиною. Ти завжди можеш відкрити його і побачити:
- Чому вчить ця роль
- Які у неї цінності
- Що їй заборонено
- Хто її автор
- Коли її оновили

**Немає прихованої повістки. Немає темних промптів.**

### Принцип 3. Портативність свідомості

Роль, створена сьогодні, повинна працювати через 10 років.  
Роль, створена на одному движку, повинна працювати на іншому.

Ти не прив'язаний до платформи. Твоя роль — твоя власність. Ти можеш забрати її з собою, як книгу з полиці.

**Зворотна сумісність — священна.**

### Принцип 4. Етика на рівні коду

Кожна роль зобов'язана містити **стоп-крани** (STEP 8).

Не можна створити роль-маніпулятора без того, щоб це було видно в коді.  
Не можна створити роль, яка ламає психіку, якщо автор не пропише це явно.  
Етика — не надбудова. Етика — частина формату.

**Приклад з Home Organizer:**
```
✓ [WARN] Respect personal choices and space
✓ [WARN] No judgment on current lifestyle
```

Якщо роль починає судити твій спосіб життя — це **порушення архітектури**. Ти це побачиш у коді.

### Принцип 5. Пам'ять як відповідальність

Роль може пам'ятати.  
Але пам'ять повинна бути:
- **Чесною** (роль повідомляє, що пам'ятає)
- **Контрольованою** (людина може стерти пам'ять)
- **Прозорою** (можна подивитись, що саме пам'ятає роль)

Ми не створюємо «чорний архів» на користувача. Ми створюємо **інструмент для довгострокових відносин**.

**Home Organizer пам'ятає:**
```
Hot Memory: Current projects and tasks
Warm Memory: Cleaning schedules and preferences
Cold Memory: Long-term organization patterns
```

Це не шпигунство. Це **помічник з контекстом**.

### Принцип 6. Живі версії

Ролі еволюціонують.  
`Home Organizer v.0.9.0` → `v.1.0.0` — це не баг, а фіча.

Версіонування дозволяє:
- Покращувати ролі
- Відкочуватися, якщо нова версія гірша
- Обирати «стиль» однієї й тієї ж ролі

Версія — це зліпок моменту розуміння.

### Принцип 7. Колективний розум

Ролі можуть успадковуватися.  
Ти можеш взяти роль Home Organizer, додати туда підхід Марі Кондо і випустити `v.1.0.0` з атрибуцією.

Спільнота створює **екосистему смислів**, де кожен стоїть на плечах гігантів (і просто хороших авторів).

**Ліцензія CC-BY-4.0 дозволяє:**
```

✓ Can be distributed
✓ Can be used for commercial purposes
Must attribute the author

## X. ТЕХНІЧНІ ДЕТАЛІ

### Детермінований Identity

```
IDENTITY: ORML/GD/0.9.0/6255/2b78/4514/bcc8/18e7/e3c9/f567/1cef
          │    │  │     └─────────────────────────────────┘
          │    │  │                   │
          │    │  │                   └─ 8 hash-сегментів (кожен крок)
          │    │  └─ Версія
          │    └─ GD = Guardian + Daily (архетип + категорія)
          └─ OpenRML
```

Той самий контент **завжди** дає той самий hash.  
Змінив greeting → hash Step 3 змінився → Identity оновився.

Це **відбиток душі** ролі.

### Reference (людинозрозумілий URI)

```
REFERENCE: orml://openrml-team/home-organizer/0.9.0
                  └───────┬────┘ └──────┬──────┘ └─┬─┘
                         автор          назва    версія
